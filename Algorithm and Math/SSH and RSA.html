# SSH란

SSH란 원격 컴퓨터간 암호화된 연결을 위한 네트워크 프로토콜이라고 한다. 즉 SSH 프로토콜 통해 검증된 사용자(내) 가 집에 있는 컴퓨터에 접속해 Shell을 이러쿵 저러쿵 할 수 있다는 뜻이다. SSH는 Public Key와 Private Key를 사용하는 비대칭 암호화 방식이다.

비대칭 암호화 방식: A가 B에게 비밀스럽게 건네줄 정보가 있다. B는 Public Key와 Private Key를 생성한 후, 모두에게 Public Key를 알려 준다. A는 원문을 Public Key로 암호화하여 B에게 보낸다. B는 Private Key로 복호화 하여 정보를 수신한다. Private Key가 유출되지 않는 이상 복호화가 어렵다.

# RSA

  1. 서로 다른 소수 p, q를 구한다.
  2. N = pq
  3. ϕ(N) = ϕ(p)ϕ(q) = (p-1)(q-1)
  4. ϕ(N)보다 작고 ϕ(N)과 서로소인 e를 찾는다.
  5. d**e ≡ 1 (MOD ϕ(N))을 만족하는 d를 찾는다.(d는 p MOD ϕ(N)의 모듈러 역원)

m = (암호화되지 않은 원문), c = (암호화된 메시지)라고 할때, 다음과 같이 암호화복호화가 이루어진다.

  • 암호화: c = m^e MOD N
  • 복호화: m = c^d MOD N

보안을 위해 p, q는 폐기한다.

# Github 레포지토리에 SSH Keys 등록하기

다음 명령어를 통해 현재 생성된 키가 있는지 확인 할 수 있다. 근데 이거 읽을 정도면 당연히 키가 없을 것이다.

ls -al ~/.ssh

ssh-keygen을 입력하여 키를 만든다.

  • -b: 키의 Bit 수다. 2048 Bits 정도면 RSA Key에 충분하다고 한다.
  • -c: 하고 싶은 말을 쓴다. "집가고 싶다."

ssh-keygen -t rsa -b 2048 -C "Your Comment"

이제 생성된 키를 ssh-agent에 등록해줘야 한다.

eval “$(ssh-agent -s)

ssh-add ~/.ssh/id_rsa

~/.ssh에 id_rsa, id_rsa.pub가 있을 것이다. 각각 Private Key와 Public Key다. id_rsa.pub의 내용을 복사하여 github의 레포지토리 settings에 들어가, Deploy keys - SSH Keys에 붙여넣기 해준다. 나는 실수로 비밀 키를 입력했더니 깃허브에서 알아서 오류를 내뿜었다.

다음을 입력하여 테스트 가능

ssh -T git@github.com
